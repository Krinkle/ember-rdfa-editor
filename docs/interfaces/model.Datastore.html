Interface Datastore

High-level interface to query RDF-knowledge from the document. Designed with the principles of a fluent interface in mind.

There are two method types: transformers and consumers. Transformers operate on the underlying knowledge (e.g. filtering triples) and always return a datastore, allowing easy method chaining.

Consumers are methods that return anything other than a datastore, typically representing the transformed knowledge in some convenient format.
