addHint

  • addHint(hrIdx: Object, who: string, card: { card: string; info: Object; location: any[]; options: undefined | { noHighlight: undefined | boolean } }): void
    • Defined in addon/utils/rdfa/hints-registry.js:514

    Adds an individual hint to the HintsRegistry.

    method

    addHint

    Parameters

    • hrIdx: Object

      Index representing the state of the registry on which you want to act. This allows the hintsRegistry to update your request with respect to user-interaction which occurred in the meantime.

    • who: string

      Unique id of the plugin for which to interact. This ID is a scope for adding and removing hints so a plugin doesn't accidentally override another plugin's cards. A common unique ID to use is the name of the plugin card (see card.card) or the name of the service.

      Eg: editor-plugins/dbpedia-info-plugin

    • card: { card: string; info: Object; location: any[]; options: undefined | { noHighlight: undefined | boolean } }

      Hint card to add

      • card: string

        Name of the component to render. This is commonly the path to your ember component in the source-tree.

        Eg: editor-plugins/dbpedia-info-card.

      • info: Object

        Information object supplied to the component in the @info argument when rendering the hint card. Anything your component needs should be supplied through this info object.

        Eg: For plugins which manipulate the document you'll want to pass the hrIdx, editor interface and hintsRegistry plus maybe some other information detected earlier such as a matched term.

      • location: any[]

        Array indicating the start and end index of the highlight in the document.

        Eg: [4,52]

      • options: undefined | { noHighlight: undefined | boolean }

    Returns void

removeHintsInRdfaBlocks

  • removeHintsInRdfaBlocks(rdfaBlocks: any[], hrId: any, identifier: undefined | string): void
    • Defined in addon/utils/rdfa/hints-registry.js:383

    Removes hints which occur in a set of rdfaBlocks.

    This is a helper method which makes functional plugins a lot easier to write. These plugins remove all hints as a starting point, calculate where the hints should be, and supply all of the new hints to the HinstRegistry. In case there are no performance issues to recalculate the hints, this approach is easy to understand. This particular call cleans up such plugins.

    method

    removeHintsInRdfaBlocks

    Parameters

    • rdfaBlocks: any[]

      Objects for which the region is in the region property.

    • hrId: any
    • identifier: undefined | string

    Returns void
